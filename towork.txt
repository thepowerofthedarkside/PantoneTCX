Задание для Codex: создать сервис «Генератор палитры коллекции» (MVP → V1)

Цель
Реализовать веб-сервис, который по параметрам коллекции генерирует структурированную палитру (5–7 цветов) с ролями (акцент/поддержка/база/нейтраль), проверками сочетаемости в LAB/LCH, процентным распределением и экспортом отчёта A4 в PDF. Сервис должен быть применим к производству: выдавать цветовые коды (HEX/RGB/CMYK/LAB) и “Pantone-аналоги” (через подключаемый справочник).

Ограничения и принципы

Это не “рандомные красивые цвета”, а палитра-система с ролями и проверками.

Расчёты в LAB/LCH, метрика близости — ΔE00.

Данные Pantone: сделать модуль “справочник” с интерфейсом и заглушкой (CSV/JSON), без нелегального вшивания данных. Поддержать подключение лицензированных данных позже.

Код читаемый, типизированный, тестируемый, без сетевых вызовов (кроме раздачи сервиса).

Функциональность MVP (обязательная)

1.1. Ввод параметров (UI + API)
Поля:

season: "SS" | "FW"

audience: "mass" | "middle" | "premium"

style: "minimal" | "street" | "romantic" | "classic" | "sport"

geography: строка (например "EU", "RU", "US") — пока как категория/тэг

key_color: optional (HEX, например "#D26C31")

sku_count: int (например 20..500)

1.2. Генерация палитры (6 цветов по умолчанию)
Выход: 6 цветов с ролями:

1 accent (10–15%)

2 supporting (20–30%)

2 base (30–40%)

1 neutral (20%)
(проценты должны суммироваться в 100%, допускается авто-нормализация)

Каждый цвет возвращать в форматах:

HEX

RGB

CMYK (приближенно, документировать, что это расчетная конвертация)

LAB (D65)

роль и рекомендованный процент

1.3. Проверка сочетаемости (обязательная)
Посчитать:

pairwise ΔE00 матрицу

разницу по L* (контраст по светлоте)
Проверки/правила:

запрет дублей: если ΔE00 < 5 между любыми двумя — считать дубликатом и перегенерировать один из них

базовые цвета должны иметь “совместимость” с большинством остальных: например средний ΔE00 в диапазоне [10..40] (настроечные пороги)

нейтрали должны иметь низкую chroma и покрывать минимум 2 уровня L* в V1 (в MVP — один нейтраль достаточно)

1.4. Pantone-аналоги (обязательная заглушка)
Сделать интерфейс ColorLibraryProvider:

list_entries() -> [{code, name, lab}]

match_top_k(lab, k=3) -> [{code, name, delta_e00}]
Реализация по умолчанию: загрузка из локального файла data/pantone_stub.csv (10–50 строк демонстрационных данных).
Важно: провайдер должен быть заменяемым на лицензированный источник без переписывания остального кода.

1.5. PDF A4 (обязательная)
Экспорт одного листа A4:

заголовок (параметры генерации)

6 плашек цветов: роль, HEX, LAB, Pantone top-1 + ΔE00

блок “рекомендованное распределение %” (простая диаграмма полосами)

примечание: “цветовые значения расчетные, зависят от среды/материала”

Алгоритм генерации (минимально-реалистичный)

Использовать LAB/LCH профиль, зависящий от входа:

season влияет на диапазон L* (SS светлее, FW темнее)

audience влияет на chroma (premium ниже/сдержаннее, mass выше/ярче)

style влияет на hue spread (minimal узкий, street широкий, romantic мягче по контрасту)

Если key_color задан:

парсить HEX -> RGB -> LAB

фиксировать как accent (или если слишком “серый”, поднять chroma/сместить hue)

Генерация остальных:

supporting: hue ± 20–40° (аналоговые/сплит-комплементарные), chroma ниже акцента

base: рядом по hue, но chroma ниже, L* ближе к середине

neutral: очень низкая chroma, L* выбран по сезону

После генерации:

прогнать правила дублей/конфликтов и при необходимости перегенерировать проблемные элементы до max_iter=20

Тесты (обязательная часть)
Минимум 10 тестов:

парсинг HEX и конвертации RGB<->LAB (проверки на диапазоны)

ΔE00 симметричность и 0 на одинаковых цветах

генерация возвращает ровно count цветов и роли распределены корректно

проценты суммируются в 100 (с учетом округления)

нет дублей: min_delta_e00 >= 5

API /api/palette/generate возвращает валидный JSON по схеме

PDF эндпоинт возвращает application/pdf и непустое тело
